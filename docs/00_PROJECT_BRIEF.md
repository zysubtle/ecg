# Project Brief v0.1｜桌面端 ECG 分析算法项目

## 1. 项目目标

本项目目标是开发一个桌面端 ECG 分析工具，用于：

- 从 BIDMC CSV 数据中读取 ECG 与 PPG；
- 对 ECG、PPG 和时间戳同步降采样到 50 Hz；
- 基于 ECG 检测 R 峰；
- 输出 RR；
- 在最小 GUI 中显示 ECG 波形与 R 峰位置；
- 作为研究 / 工程分析工具使用，不作为医疗诊断软件。

## 2. 项目非目标

当前阶段不做：

- 医疗诊断结论；
- 临床级准确性声明；
- HRV 指标计算；
- PPG IBI 计算；
- 多导联融合；
- 复杂 GUI；
- 批量处理；
- 人工编辑 R 峰；
- 完整商业发布打包。

## 3. 输入数据与字段

已确认：

- 数据来源：BIDMC；
- 输入格式：CSV；
- 原始采样率：125 Hz，固定；
- 时间戳字段：`Time [s]`，单位秒；
- ECG 导联字段：` II`；
- PPG 字段：`PLETH` / ` PLETH`；
- 当前只处理一个 ECG 导联；
- 字段名可能包含前导 / 后缀空格，因此读取时需要支持 `strip()` 后匹配。

## 4. 输出结果与字段

已确认导出字段包括：

- 降采样后的时间戳；
- 降采样后的 ECG；
- 降采样后的 PPG；
- `r_peak_sample_index`；
- `r_peak_time_s`；
- `rr_ms`；
- `detector_name`；
- `quality_flag`。

当前不导出：

- `hr_bpm`；
- HRV 指标；
- 医疗解释字段。

已确认默认假设：主输出 CSV 按 50 Hz 采样点逐行输出，R 峰相关字段只在对应 R 峰行填充，非 R 峰行为空。

## 5. ECG 算法范围

当前算法范围：

- 读取 ECG；
- 降采样；
- R 峰检测；
- RR 计算；
- 输出 R 峰位置和 RR。

已确认默认假设：优先在原始 125 Hz ECG 上检测 R 峰，然后把 `r_peak_time_s` 映射到 50 Hz 输出索引。原因是 125 Hz 检测 R 峰比 50 Hz 检测有更高时间分辨率。

## 6. 第三方库 / 自研策略

已确认：

- 允许使用第三方 ECG R 峰 / QRS 检测库；
- 允许 Codex 安装 pip 依赖；
- 无 license 限制；
- 不要求支持商业闭源使用；
- 不保留自研 fallback；
- 需要先做 **M2a：第三方 R 峰检测库评估与选型**。

候选库包括但不限于：

- NeuroKit2；
- WFDB；
- BioSPPy；
- SleepECG；
- HeartPy；
- scipy / numpy signal processing 组合方案。

策略原则：

- 不默认从零自研 R 峰检测算法；
- M2a 前不得锁定具体第三方库；
- 第三方库必须通过 adapter 封装；
- GUI、CLI、导出模块不得直接依赖具体第三方库 API；
- license、维护风险、安装复杂度、桌面端打包风险均在 M2a 中评估。

## 7. 桌面端功能范围

已确认：

- 需要 CLI；
- 需要最小 GUI；
- GUI 需要显示 ECG 波形与 R 峰位置。

当前 GUI 范围建议：

- 选择 CSV；
- 运行分析；
- 显示降采样 ECG 波形；
- 标记 R 峰；
- 显示基础结果；
- 支持导出 CSV。

复杂交互暂不做。

## 8. 技术栈与平台默认假设

已确认默认假设：

- 使用 Python 作为主技术栈；
- CLI + 最小 GUI；
- 使用 pip 管理依赖；
- 后续根据 M2a 结果确定具体 ECG detector 依赖；
- GUI 框架暂不锁死，后续里程碑再确认；
- 示例 CSV 作为仓库内 fixture 使用。

## 9. 测试与验收标准

当前验收重点：

- 能读取 BIDMC CSV；
- 能正确识别必要字段；
- 能处理字段名前后空格；
- 能验证采样率约为 125 Hz；
- 能将 ECG、PPG、时间戳同步降采样到 50 Hz；
- 能检测 R 峰；
- 能输出 RR；
- 能导出字段完整的 CSV；
- GUI 能显示 ECG 波形和 R 峰；
- 示例 CSV 只作为 smoke test / 回归测试，不作为总体准确性证明。

当前没有参考标注，因此暂不做定量准确率结论。

## 13. 风险清单

当前主要风险：

- **S0 风险**：最终第三方库选型必须经过 M2a 后由 Owner 决策；
- **S1 风险**：字段名前后空格若处理不好，会导致 CSV 读取失败；
- **S2 风险**：单个 BIDMC 示例 CSV 只能做 smoke test，不能证明总体准确性；
- **S2 风险**：不同第三方库的 R 峰输出格式、依赖复杂度、打包难度可能不同；
- **S2 风险**：无参考标注时，只能做人工抽查和工程合理性检查。

## 14. 里程碑草案

- **M0：项目启动问诊**  
  已完成。

- **M1：项目启动包**  
  生成项目文档、协作规则、IO Contract、Milestone Plan、Codex Runbook、当前任务文件和最小 Python 项目骨架。

- **M2a：第三方 R 峰检测库评估与选型**  
  至少评估 2-3 个候选库，并在示例 CSV 上做 smoke test，形成 Owner 可决策方案。

- **M2b：数据读取与降采样管线**  
  实现 CSV 读取、字段归一化、125 Hz 校验、ECG / PPG / 时间戳降采样到 50 Hz。

- **M3：Detector Adapter 集成与 RR 输出**  
  按 Owner 选定的 detector 实现统一 adapter，输出 R 峰和 RR。

- **M4：CLI 与导出结果完善**  
  完成命令行入口、输出 CSV、错误处理和最小回归测试。

- **M5：最小 GUI 与波形可视化**  
  选择 CSV、运行分析、显示 ECG 波形和 R 峰。

- **M6：验收、文档收敛与风险复盘**  
  汇总测试结果、限制条件、后续优化建议。
